# 老虎機數學入門 PART Ⅳ

來源：https://ezslotdesign.com/slotmath4/

這篇主要討論的是 **Way Game** 的 RTP 計算，以及它和 **Line Game** 的差異。

PART I 的 Line Game 是「針對一條線」計算；PART IV 的 Way Game 則是「針對整個畫面」計算。

在 3 x 5 的盤面中，Way Game 常被稱為：

```text
243 Ways = 3 x 3 x 3 x 3 x 3
```

但重點是：

```text
Way Game 不等於 243 條 Line Game 直接相加
```

因為兩者的計分方式不同。

---

## Line Game vs Way Game

Line Game 是看「指定線」上的圖案。

例如一條線穿過每一軸的中間格，那只有落在線上的圖案會參與計算。

Way Game 則是看「整個畫面」。

只要由左到右相鄰滾輪出現同一個圖案，不管它出現在該輪的哪一格，都可以形成 Way。

例如某圖案在前四軸出現：

```text
第 1 軸：1 個
第 2 軸：3 個
第 3 軸：2 個
第 4 軸：1 個
```

若該圖案是 4 連線，賠率為 500，則派彩不是單純 500，而是：

```text
500 x 1 x 3 x 2 x 1 = 3000
```

也就是：

```text
Way 派彩 = 連線賠率 x 各軸畫面上命中的個數
```

---

## Way Game 只算最長連線

文章中特別提醒：

Line Game 中，多條線可能同時中獎，會分別計分。

但 Way Game 通常對同一個圖案只看從左到右的最長連線。

例如：

```text
同一圖案同時可看成 2 連線與 3 連線
```

Way Game 會算最長的 3 連線，而不是 2 連線和 3 連線都算。

所以 Way Game 的計算重點是：

```text
由左到右連續到第幾軸
每一軸有幾個可形成該圖案的 stop
```

---

## 為什麼圖案次數要乘上視窗高度

在 3 x 5 Way Game 中，每個滾輪畫面有 3 格高度。

如果某圖案在輪帶上出現 1 次，當滾輪停在不同位置時，這個圖案可能出現在畫面的上、中、下三格。

所以在 Way Game 裡，一個輪帶上的圖案出現 1 次，會對應到 3 種畫面位置。

因此文章中的 `WILD + 各圖案個數` 會寫成：

```text
WK = (W 的輪帶個數 + K 的輪帶個數) x 轉輪高度
```

若轉輪高度是 3：

```text
WK = (W + K) x 3
```

例如某軸：

```text
W = 1
K = 3
```

則：

```text
WK = (1 + 3) x 3 = 12
```

這代表在 Way Game 的視窗中，W 或 K 可以出現在畫面任一格，所以要乘上高度。

---

## X_WK 不能用簡單扣法

這篇最容易踩坑的地方是 `X_WK`。

在 Line Game 中：

```text
X_WK = 不是 W，也不是 K 的位置數
```

可以簡單算：

```text
X_WK = 全部位置數 - W個數 - K個數
```

但在 Way Game 中，不能直接寫成：

```text
X_WK = 輪帶長度 - (W + K) x 3
```

原因是：如果圖案在輪帶上相鄰或堆疊，乘以 3 之後會重複扣除同一個畫面 stop。

文章舉的概念是：

如果某圖案 Q 在輪帶上有 2 個，而且相鄰堆疊，直覺可能會算：

```text
10 - 2 x 3 = 4
```

但實際上，Q 會出現在畫面的可能 stop 可能只有 4 種，所以：

```text
X_WQ = 10 - 4 = 6
```

不是 4。

---

## 正確的 X_WK 算法

Way Game 的 `X_WK` 要逐 stop 檢查。

假設滾輪高度是 3，對每一個停輪位置，都看畫面中的 3 格：

```text
位置 i
位置 i+1
位置 i+2
```

如果這 3 格中有出現 W 或 K，代表這個 stop 會讓畫面出現 W/K，因此不是 `X_WK`。

如果這 3 格完全沒有 W，也沒有 K，才算 1 個 `X_WK`。

所以：

```text
X_WK
= 畫面中完全沒有 W/K 的 stop 數
```

或等價地：

```text
X_WK
= 輪帶長度 - 畫面中有出現 W/K 的 stop 數
```

這就是文章說的「實際未出現」。

---

## 本 Excel 的 X_WK 公式寫法

右側會建立類似這樣的判斷表：

```text
X_WK 判斷表

stop | 轉輪1 | 轉輪2 | 轉輪3 | 轉輪4 | 轉輪5
1    | 1     | 1     | 1     | 1     | 1
2    | 0     | 1     | 1     | 1     | 1
3    | 0     | 0     | 0     | 0     | 0
...
20   | 1     | 1     | 1     | 1     | 1
```

每一格的意思是：

```text
1 = 該 stop 的 3 格視窗中，有 W 或該圖案
0 = 該 stop 的 3 格視窗中，完全沒有 W/該圖案
```

例如在 `X_WK` 判斷表中，某一格是 1，代表該 stop 的 3 格視窗中有 W 或 K。

某一格是 0，代表該 stop 的 3 格視窗中完全沒有 W/K。

每個 stop 的判斷公式概念是：

```excel
=IF(COUNTIF(該 stop 的 3 格視窗,"W")+COUNTIF(該 stop 的 3 格視窗,"K")>0,1,0)
```

例如 stop 1 的視窗是：

```excel
B53:B55
```

則判斷是否有 W/K：

```excel
=IF(COUNTIF(B53:B55,"W")+COUNTIF(B53:B55,"K")>0,1,0)
```

主統計表中的 `X_WK` 會直接引用這張判斷表：

```excel
=COUNTIF(X_WK判斷欄,0)
```

也就是直接數：

```text
完全沒有 W/K 的 stop 有幾個
```

這和文章的想法一致：

```text
文章：逐 stop 判斷，有 W/K 記 1，沒有 W/K 記 0，最後數 0
本 Excel：用 X_WK 判斷表逐 stop 判斷，主表用 COUNTIF(...,0) 數 0
```

因此現在 Excel 裡的 `X_WK / X_WQ / X_WJ / X_W10 / X_W9` 都是用文章式輔助表算出來的。

## 計算流程

文章的 Way Game RTP 計算可以拆成以下步驟。

### 1. 統計輪帶上的原始圖案個數

例如：

```text
W, K, Q, J, 10, 9, S
```

在每一軸各出現幾次。

這是最基礎的統計表。

### 2. 計算 Wild + 各圖案個數

對一般圖案，例如 K：

```text
WK = (W + K) x 轉輪高度
```

Q、J、10、9 也同理：

```text
WQ  = (W + Q)  x 轉輪高度
WJ  = (W + J)  x 轉輪高度
W10 = (W + 10) x 轉輪高度
W9  = (W + 9)  x 轉輪高度
```

這些值代表該圖案在 Way Game 畫面中的可命中個數。

### 3. 計算實際未出現 X_W*

例如：

```text
X_WK
X_WQ
X_WJ
X_W10
X_W9
```

這些不能用簡單扣法，而要逐 stop 檢查視窗。

### 4. 列出中獎 partition

和 Line Game 類似，仍然可以列成：

```text
K 5連：WK, WK, WK, WK, WK
K 4連：WK, WK, WK, WK, X_WK
K 3連：WK, WK, WK, X_WK, -
```

其他圖案同理。

Scatter 若用簡化方式，也可以列成：

```text
S 3連：-, 3S, 3S, 3S, -
```

### 5. 計算出現次數

每一列的出現次數仍然是各軸相乘：

```text
出現次數
= 轉輪1條件數
 x 轉輪2條件數
 x 轉輪3條件數
 x 轉輪4條件數
 x 轉輪5條件數
```

例如：

```text
K 5連
= WK1 x WK2 x WK3 x WK4 x WK5
```

### 6. 計算出現機率

```text
出現機率 = 出現次數 / 所有可能數
```

如果每軸長度都是 20：

```text
所有可能數 = 20^5 = 3,200,000
```

### 7. 計算期望值

Way Game 的期望值要除以本把下注 coin。

文章範例使用：

```text
coin = 50
```

所以：

```text
期望值
= 出現機率 x 得分數 / coin
```

最後：

```text
總 RTP = 所有 partition 的期望值加總
```

---

## 和 Line Game 的最大差異

Line Game 的核心是：

```text
單線上的一個位置是否命中
```

Way Game 的核心是：

```text
整個畫面中該圖案出現幾個
```

因此 Way Game 需要特別處理兩件事：

```text
1. 命中數要乘上視窗高度
2. 未出現數要用實際畫面 stop 判斷，不能簡單相減
```

這兩點是 PART IV 的核心。

---

## 總結

PART IV 的重點可以濃縮成：

```text
Way Game 是整個畫面的期望值計算，不是單線計算。
```

計算上：

```text
WK = (W + K) x 視窗高度
```

但：

```text
X_WK != 輪帶長度 - WK
```

而是：

```text
X_WK = 畫面視窗中實際沒有 W/K 的 stop 數
```

最後每個 partition：

```text
期望值 = 出現次數 / 所有可能數 x 得分數 / coin
```

將所有 partition 的期望值加總，就是 Way Game 的 RTP。

---

## 補充：Scatter 和 Way Game 的計數差異

這裡容易混淆的是：

```text
Scatter 的計分方式
Way Game 一般圖案的計分方式
```

兩者都會受到「視窗高度」與「相鄰圖案」影響，但它們真正要算的量不一樣。

---

### Way Game：出現顆數是倍數

以 K 為例，Way Game 要算的是：

```text
每一軸畫面中有幾顆 K / W
```

這個數字會直接變成該軸的 ways multiplier。

例如某次畫面：

```text
第 1 軸有 1 顆 K
第 2 軸有 3 顆 K
第 3 軸有 2 顆 K
```

那 K 3 連線的 ways 數就是：

```text
1 x 3 x 2 = 6 ways
```

所以如果輪帶上有兩顆 K，即使它們相鄰，每一顆 K 仍然都可以出現在畫面的上、中、下三格。

因此：

```text
K 的畫面出現貢獻 = K 個數 x 視窗高度
```

如果：

```text
K 個數 = 2
視窗高度 = 3
```

則：

```text
K 的畫面出現貢獻 = 2 x 3 = 6
```

相鄰不會讓這個值變小，因為 Way Game 需要的是「畫面中總共有幾顆可用圖案」，不是「有幾個 stop 會看到它」。

所以：

```text
WK = (W + K) x 視窗高度
```

這是在算 ways multiplier 的總貢獻。

---

### Scatter：看畫面總顆數

Scatter 通常不是拿來做 ways multiplier，而是看整個畫面總共有幾顆 Scatter。

例如：

```text
畫面中有 3 顆 S -> 3S 獎 / 觸發
畫面中有 4 顆 S -> 4S 獎 / 觸發
畫面中有 5 顆 S -> 5S 獎 / 觸發
```

因此 Scatter 的問題是：

```text
這個 stop 畫面中看到幾顆 S？
```

如果一軸上有兩顆相鄰 S，某些 stop 的 3 格視窗可能會同時看到兩顆 S。

所以 Scatter 不能只用：

```text
S 個數 x 視窗高度
```

因為我們需要知道每個 stop 對應的是：

```text
看到 0 顆 S
看到 1 顆 S
看到 2 顆 S
看到 3 顆 S
```

接著再把五軸組合起來，計算整個畫面總 Scatter 數等於 3、4、5 的出現次數。

也就是：

```text
Scatter 要保留「每個 stop 看到幾顆」的分布
Way Game 一般圖案只需要「畫面中出現幾顆」作為倍數
```

---

### X_WK 又是另一種問題

`X_WK` 雖然出現在 Way Game 的一般圖案計算裡，但它不是在算顆數。

它算的是：

```text
這個 stop 的畫面中是否完全沒有 W/K
```

這是一個 yes/no 問題。

如果某個 stop 的畫面是：

```text
K
K
Q
```

雖然畫面中有兩顆 K，但對 `X_WK` 來說只代表：

```text
有 W/K -> 這個 stop 不算 X_WK
```

不會因為看到兩顆 K 就扣兩次。

所以 `X_WK` 必須逐 stop 檢查視窗：

```text
stop i 的 3 格視窗中是否完全沒有 W/K？
```

最後：

```text
X_WK = 完全沒有 W/K 的 stop 數
```

---

### 一句話整理

```text
Way Game 一般圖案：
算每軸畫面中有幾顆，顆數直接當 multiplier。

Scatter：
算整個畫面總共有幾顆，因此要保留每個 stop 看到幾顆的分布。

X_WK：
算某個 stop 是否完全沒有 W/K，是 yes/no 的斷線判斷。
```
