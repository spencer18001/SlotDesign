# 黏性百搭 Sticky Wild

Source: https://ezslotdesign.com/stickywild/

## 核心概念

Sticky Wild（黏性百搭，也常叫 Lock Wild）是 Free Game 中常見的百搭特色。

規則是：

```text
Wild 出現在某個位置後，會黏在該位置不動，直到 Free Game 結束。
```

因此 Free Game 越往後，盤面上累積的 Wild 通常越多，連線期望值也會逐步上升。

這類特色的數學重點不是只算單一 spin 的 RTP，而是要追蹤：

```text
目前哪些位置已經被 Wild 黏住？
```

這是一個「狀態會延續」的問題，所以可以用馬可夫鏈或狀態分布的觀念處理。

## 狀態怎麼定義

文章先把一條得分線上的 5 個位置，各自視為兩種狀態：

```text
0 = 該位置還沒有 Sticky Wild
1 = 該位置已經有 Sticky Wild 黏住
```

一條 5 軸 line 共有 5 個位置，所以總狀態數是：

```text
2^5 = 32 種狀態
```

例如：

```text
00000 = 5 個位置都還沒有黏住 Wild
10000 = 第 1 軸位置已經黏住 Wild
11101 = 第 1,2,3,5 軸位置已經黏住 Wild，第 4 軸還沒有
11111 = 5 個位置都已經黏住 Wild
```

這裡的狀態只描述「哪些位置已經黏住 Wild」，不是描述當下新轉出的整個盤面。

## 每個狀態先算自己的 RTP

對 32 種狀態，每一種都可以算一個對應 RTP。

做法是：

```text
如果某位置已經是 Sticky Wild，
就把該位置視為固定 Wild，
再計算該狀態下的一條線 RTP。
```

例如狀態：

```text
10100
```

表示第 1 軸與第 3 軸已經固定是 Wild。

那計算該狀態 RTP 時，可以把第 1 軸與第 3 軸的該 line 位置直接當成 Wild，其餘軸照正常輪帶機率計算。

因此整體流程第一步是：

```text
State RTP table:
state 00000 -> RTP_00000
state 00001 -> RTP_00001
...
state 11111 -> RTP_11111
```

## 每一把 Free Spin 的狀態分布

算完每個狀態 RTP 後，還要知道：

```text
第 n 把 Free Spin 開始時，各狀態出現的機率是多少？
```

文章用「單一位置還沒黏住的機率」來推。

假設某一軸某一格，在一把 Free Spin 之後沒有黏住 Wild 的機率是：

```text
q = P(沒黏住率)
```

那麼第 n 把 Free Spin 開始時，這個位置仍然沒被黏住，代表前面 n-1 把都沒有黏住：

```text
P(第 n 把開始時仍未黏住) = q^(n-1)
```

相反地，該位置已經黏住的機率是：

```text
P(第 n 把開始時已黏住) = 1 - q^(n-1)
```

文章中也可寫成：第 n 把結束後或第 n+1 把開始時：

```text
未黏住 = q^n
已黏住 = 1 - q^n
```

兩種寫法只是時間點不同：

```text
第 n 把開始 = 已經過 n-1 把
第 n+1 把開始 = 已經過 n 把
```

## 狀態機率怎麼算

一個狀態的機率，就是把 5 個位置各自「已黏住 / 未黏住」的機率相乘。

例如狀態：

```text
11101
```

代表：

```text
位置 1 已黏住
位置 2 已黏住
位置 3 已黏住
位置 4 未黏住
位置 5 已黏住
```

如果第 n 把開始時，各位置已黏住機率分別是：

```text
s1, s2, s3, s4, s5
```

未黏住機率就是：

```text
1-s1, 1-s2, 1-s3, 1-s4, 1-s5
```

那狀態 `11101` 的機率是：

```text
P(11101)
= s1 × s2 × s3 × (1-s4) × s5
```

文章舉的狀態 29，大意就是：某些軸已經有 Wild 黏住，某些軸還沒有，將這 5 個位置的機率相乘。

## 用 q 表示更簡潔

若每個位置每一把沒有黏住 Wild 的機率相同，都是 q，則第 n 把開始時：

```text
未黏住機率 = q^(n-1)
已黏住機率 = 1 - q^(n-1)
```

如果某狀態有 k 個位置已黏住、5-k 個位置未黏住，且五個位置的 q 相同，則該狀態機率可以簡化為：

```text
P(state at spin n)
= (1 - q^(n-1))^k × (q^(n-1))^(5-k)
```

但如果不同軸、不同位置的 Wild 出現率不同，就不能只看 k 個已黏住，而要逐位置相乘：

```text
P(state)
= Π P(該位置符合 state 要求的狀態)
```

## Free Game RTP 怎麼算

對第 n 把 Free Spin，已知：

```text
P_n(state) = 第 n 把開始時，該 state 的機率
RTP(state) = 該 state 下，一把 spin 的 RTP
```

則第 n 把 Free Spin 的期望值是：

```text
EV_n = Σ_state P_n(state) × RTP(state)
```

整個 Free Game 如果有 N 把，則 Free Game RTP 是：

```text
Free Game RTP
= EV_1 + EV_2 + ... + EV_N
= Σ_n Σ_state P_n(state) × RTP(state)
```

也就是文章最後說的：

```text
有了 32 種狀態的 RTP
以及每一把 Free Spin 中 32 種狀態的機率分布
兩者對應相乘再加總
即可得到整個 Free Game RTP
```

## 和馬可夫鏈的關係

Sticky Wild 本質上是一個狀態轉移問題。

狀態是：

```text
哪些位置已經黏住 Wild
```

狀態只會往「更多 Wild 黏住」的方向前進，不會倒退。

例如：

```text
00000 -> 10000 -> 10100 -> 11100
```

但不會：

```text
10100 -> 00100
```

因為已經黏住的 Wild 不會消失。

如果用完整馬可夫鏈寫法，可以建立：

```text
M[i,j] = P(next state = i | current state = j)
```

其中：

```text
j = 目前已黏住的位置集合
i = 下一把後已黏住的位置集合
```

而且只有當 i 包含 j 時，轉移機率才可能大於 0。

```text
如果 next state 少了原本已黏住的位置，機率 = 0
```

但文章使用的是比較簡化的方式：

```text
直接算第 n 把開始時，每個位置已黏住 / 未黏住的機率，
再組合成 32 種 state 的機率分布。
```

這在 Sticky Wild 的規則單純、各位置黏住事件可拆開處理時很直觀。

## 為什麼不能只算平均 Wild 數

Sticky Wild 不能只算「平均有幾個 Wild」。

原因是：

```text
同樣都是 2 個 Sticky Wild，位置不同，對 line pay 的影響可能不同。
```

例如：

```text
11000 = 前兩軸是 Wild
00110 = 第三、四軸是 Wild
```

兩者都有 2 個 Wild，但對由左到右連線的幫助差很多。

所以必須保留「位置狀態」，而不只是保留「Wild 數量」。

這也是為什麼文章要列出 32 種狀態。

## Excel 實作建議

可以拆成三張表：

```text
1. State List
   列出 00000 ~ 11111 共 32 種狀態。

2. State RTP
   對每個 state 算該狀態下一把 spin 的 RTP。

3. State Probability by Spin
   對第 1 ~ N 把 Free Spin，算每個 state 的出現機率。
```

最後做 Summary：

```text
Spin n EV = SUMPRODUCT(State Probability Row, State RTP Column)
Free Game RTP = SUM(Spin n EV)
```

在 Excel 中概念會像：

```excel
=SUMPRODUCT(該把的32種狀態機率, 32種狀態RTP)
```

## 精簡重點

Sticky Wild 的計算核心是：

```text
1. 把黏住位置視為狀態。
2. 一條 5 軸 line 有 2^5 = 32 種 Sticky Wild 狀態。
3. 每個狀態先各自計算 RTP。
4. 每一把 Free Spin 開始時，計算 32 種狀態的機率分布。
5. 每把 EV = 狀態機率 × 狀態 RTP 的加權平均。
6. Free Game RTP = 各把 Free Spin EV 加總。
```

和馬可夫鏈筆記銜接起來就是：

```text
狀態 = 哪些位置已經黏住 Wild
轉移 = 新出現的 Wild 會加入狀態
吸收方向 = Wild 只會增加，不會消失
期望值 = 對每個狀態的 RTP 做機率加權
```
