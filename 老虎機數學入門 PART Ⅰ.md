# 老虎機數學入門 PART Ⅰ

這篇文章的核心是在示範如何用 Excel 計算一個簡化 Slot Game 的 RTP。範例主要拆成兩種獎項：

- 一條 Payline 上的連線獎
- 簡化型 Scatter 獎

## 基本設定

範例是 5 軸、每軸 20 格的老虎機。

```text
總可能數 = 20 * 20 * 20 * 20 * 20 = 3,200,000
```

每一種獎項都先算「出現次數」，再除以總可能數得到機率，最後乘上賠分得到期望值。

```text
出現機率 = 出現次數 / 總可能數
期望值 = 出現機率 * 賠分
RTP = 所有期望值加總
```

## Line 獎計算原理

Line 獎是指定一條線，從第 1 軸開始連續命中同一個圖案才算獎。

例如 K 的 5 連線：

```text
WK / WK / WK / WK / WK
```

這裡的 `WK` 代表：

```text
W 或 K
```

因為 Wild 可以代替 K，所以每一軸的可命中數是：

```text
該軸 W 的數量 + 該軸 K 的數量
```

如果統計表中 `WK` 是：

```text
2, 4, 4, 4, 4
```

那 K 五連線的出現次數就是：

```text
2 * 4 * 4 * 4 * 4 = 512
```

再乘上賠率表中的 K 五連線賠分，即可得到這一列的期望值。

## 3 連線與 4 連線為什麼要有 X_WK

如果計算 K 的 4 連線，條件不是「前四軸是 K/W，第五軸隨便」。

而是：

```text
WK / WK / WK / WK / X_WK
```

`X_WK` 代表：

```text
不是 W，也不是 K
```

也就是第 5 軸必須斷掉，才算剛好 4 連線，而不是 5 連線。

所以 K 四連線的出現次數是：

```text
轉輪1 WK * 轉輪2 WK * 轉輪3 WK * 轉輪4 WK * 轉輪5 X_WK
```

K 三連線則是：

```text
WK / WK / WK / X_WK / -
```

意思是：

- 前三軸要命中 K 或 Wild
- 第四軸要斷掉
- 第五軸不限

`-` 代表該軸不限，因此等於該軸總格數 20。

## 統計表中的輔助符號

文章會先整理幾種輔助統計：

| 符號 | 意思 |
|---|---|
| `WK` | W 或 K 的數量 |
| `WQ` | W 或 Q 的數量 |
| `WJ` | W 或 J 的數量 |
| `W10` | W 或 10 的數量 |
| `W9` | W 或 9 的數量 |
| `X_WK` | 不是 W，也不是 K 的數量 |
| `X_WQ` | 不是 W，也不是 Q 的數量 |
| `-` | 該軸不限，等於該軸總格數 |

右邊的得分組合表，就是把每種獎轉成五軸條件，再把五軸對應數量相乘。

## Excel 公式概念

出現次數常見公式是：

```excel
=PRODUCT(
  VLOOKUP(轉輪1條件, 統計表, 轉輪1欄位, FALSE),
  VLOOKUP(轉輪2條件, 統計表, 轉輪2欄位, FALSE),
  VLOOKUP(轉輪3條件, 統計表, 轉輪3欄位, FALSE),
  VLOOKUP(轉輪4條件, 統計表, 轉輪4欄位, FALSE),
  VLOOKUP(轉輪5條件, 統計表, 轉輪5欄位, FALSE)
)
```

意思是：每一軸依照條件去統計表查數量，最後全部相乘。

賠分則用圖案與連線數去賠率表查：

```excel
=INDEX(賠率數值區, MATCH(圖案, 圖案欄, 0), MATCH(連線數&"連線", 連線標題列, 0))
```

## 簡化型 Scatter 計算原理

Scatter 獎和 Line 獎不同。Line 獎看指定線，Scatter 通常看整個畫面出現幾個 Scatter。

這篇文章用的是簡化版本：

- 視窗高度是 3 row
- Scatter 符號是 `S`
- 每個有 S 的轉輪上，S 不會彼此重疊造成同一停格看到多個 S
- 因此可以用 `S 的個數 * 3` 近似該軸可見 Scatter 次數

例如第 2、3、4 軸各有 1 個 S，第 1、5 軸沒有 S：

```text
S:   0, 1, 1, 1, 0
3*S: 0, 3, 3, 3, 0
```

如果 Scatter 獎是 3 個 S，條件可以寫成：

```text
- / 3*S / 3*S / 3*S / -
```

意思是：

- 第 1 軸不限
- 第 2 軸要看到 Scatter
- 第 3 軸要看到 Scatter
- 第 4 軸要看到 Scatter
- 第 5 軸不限

出現次數：

```text
20 * 3 * 3 * 3 * 20 = 10,800
```

機率：

```text
10,800 / 3,200,000 = 0.003375
```

若賠分是 50：

```text
期望值 = 0.003375 * 50 = 0.16875
```

也就是 16.875%。

## 這個 Scatter 算法的限制

這篇的 Scatter 算法是簡化型，適合教學入門，但不是所有情況都能直接用。

如果同一軸有多個相鄰的 S，3-row 視窗可能一次看到 2 個甚至 3 個 S。這時就不能只用：

```text
S 個數 * 視窗高度
```

比較完整的做法應該是統計每一軸在每個停格下會看到幾個 Scatter：

```text
看到 0 個 S 的 stop 數
看到 1 個 S 的 stop 數
看到 2 個 S 的 stop 數
看到 3 個 S 的 stop 數
```

然後再組合五軸，計算畫面總 Scatter 數等於 3、4、5 的出現次數。

## 小結

這篇文章的核心不是完整商業遊戲的全部數學，而是示範 Slot Math 的基本思路：

1. 先統計每軸符號數量
2. 把每種獎項轉成五軸條件
3. 用各軸條件數量相乘得到出現次數
4. 除以總可能數得到機率
5. 乘上賠分得到期望值
6. 加總所有期望值得到 RTP

Line 獎的重點是「從第 1 軸開始連續命中，並用 X_W* 表示剛好斷線」。

Scatter 的重點是「這篇只用簡化算法，將每軸 Scatter 數量乘上視窗高度，但實務上若同軸可能看到多個 Scatter，就要改用可見 Scatter 分布計算」。

## 多條 Line 的派彩與期望值線性性

前面範例只計算一條 Payline 的 RTP。如果遊戲有多條線，而且每一條線的機率模型與賠率都相同，則總 Line RTP 可以用單線 RTP 直接乘上線數。

假設：

```text
X1 = 第 1 條線的派彩
X2 = 第 2 條線的派彩
T  = 兩條線的總派彩
```

則：

```text
T = X1 + X2
```

根據期望值的線性性：

```text
E[T] = E[X1 + X2] = E[X1] + E[X2]
```

如果兩條線的期望值相同：

```text
E[X1] = E[X2]
```

那麼：

```text
E[T] = 2 * E[X1]
```

推廣到 N 條線：

```text
Total Line RTP = Number of Lines * Single Line RTP
```

### 不需要假設各線獨立

這個結論不需要假設每條線彼此獨立。

即使兩條線共用同一顆 Wild，或因為同一個畫面而高度相關，只要總派彩是各線派彩相加，期望值仍然可以直接相加。

例如某一次 spin：

```text
第 1 條線中 100
第 2 條線也因為同一顆 Wild 中 100
總派彩 = 200
```

或另一個 spin：

```text
第 1 條線中 100
第 2 條線沒中
總派彩 = 100
```

不管兩條線之間是否相關，平均值都符合：

```text
E[X + Y] = E[X] + E[Y]
```

所以在計算多條線 Slot RTP 時，通常可以先算一條線的 RTP，再乘上線數。

### 使用前提

這個簡化有一個重要前提：每條 Payline 的期望值要相同。

如果不同線有不同形狀、不同可命中位置、不同賠率或特殊規則，則不能直接用線數乘單線 RTP，而要分別計算每條線：

```text
Total Line RTP = RTP1 + RTP2 + RTP3 + ... + RTPn
```

Scatter、Bonus、Free Game 等不屬於單條線的獎項，通常另外計算，再加到總 RTP 中。
