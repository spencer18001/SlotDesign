# 二項分配機率

Source: https://ezslotdesign.com/binomial-distribution/

## 核心概念

二項分配（Binomial Distribution）用來描述：

```text
在 n 次彼此獨立、成功機率相同的試驗中，剛好成功 k 次的機率。
```

記作：

```text
X ~ Binomial(n, p)
```

其中：

```text
n = 試驗次數
p = 每次成功機率
k = 成功次數
```

機率公式是：

$$
P(X=k)=\binom{n}{k}p^k(1-p)^{n-k}
$$

直覺上可以拆成兩部分：

```text
C(n,k)       = 從 n 次裡挑 k 次成功的排列數
p^k         = 這 k 次成功的機率
(1-p)^(n-k) = 其他 n-k 次失敗的機率
```

## 在 Slot / Bonus 遊戲中的用途

文章用一個簡化 bonus game 當例子：

```text
有 15 個箱子
每個箱子會開出 B1, B2, B3, B4, B5 其中一種圖案
各圖案機率分別是：
B1 = 10%
B2 = 15%
B3 = 20%
B4 = 25%
B5 = 30%
```

如果只關心某一種圖案，例如 B1，那麼每個箱子可以簡化成：

```text
成功 = 開到 B1
失敗 = 沒開到 B1
```

因此：

```text
X_B1 ~ Binomial(15, 0.10)
```

就可以計算：

```text
15 個箱子中，剛好開到 0 個 B1 的機率
15 個箱子中，剛好開到 1 個 B1 的機率
...
15 個箱子中，剛好開到 15 個 B1 的機率
```

Excel 可用：

```excel
=BINOM.DIST(k, 15, 0.10, FALSE)
```

其中：

```text
k     = 想算的成功次數
15    = 總箱子數
0.10  = 每個箱子開到 B1 的機率
FALSE = 計算剛好 k 次，而不是累積機率
```

## 等級獎勵：區間機率

如果獎勵不是「剛好 k 個」，而是依照累積個數給不同等級，例如：

```text
Level 1: 至少 5 個
Level 2: 至少 9 個
Level 3: 至少 12 個
Level 4: 至少 14 個
Level 5: 15 個
```

那每一級通常要算的是區間機率。

例如 B1 得到 Level 1，但還沒到 Level 2：

```text
5 <= X_B1 <= 8
```

可以寫成：

$$
P(5 \le X \le 8)
= \sum_{k=5}^{8}\binom{15}{k}p^k(1-p)^{15-k}
$$

Excel 可用：

```excel
=BINOM.DIST.RANGE(15, 0.10, 5, 8)
```

或等價地用累積機率相減：

```excel
=BINOM.DIST(8,15,0.10,TRUE)-BINOM.DIST(4,15,0.10,TRUE)
```

## 重要注意：不同圖案個數不是互相獨立

如果同一個箱子只能開出一種圖案，那麼：

```text
B1 個數多，其他圖案個數自然會變少。
```

所以不同圖案的開出個數彼此有關聯。

例如要問：

```text
B1, B2, B3, B4, B5 剛好各 3 個的機率
```

不能直接寫成：

```text
P(B1=3) × P(B2=3) × P(B3=3) × P(B4=3) × P(B5=3)
```

因為這五個事件不是獨立事件。

這種「多類別結果的個數分布」更適合用多項分配（Multinomial Distribution）：

$$
P(N_1=n_1,\ldots,N_m=n_m)
= \frac{n!}{n_1!\cdots n_m!}\prod_{i=1}^m p_i^{n_i}
$$

前提是：

```text
n_1 + n_2 + ... + n_m = n
```

## 多輪重開：狀態轉移的想法

文章第三個例子是：

```text
只收集 B1
開到 B1 的箱子會留下
不是 B1 的箱子會關起來重開
最多重複 5 輪
```

這時就不是單純一次二項分配，而是每一輪的狀態會影響下一輪。

可以把狀態定義為：

```text
目前已經收集到幾個 B1
```

例如採用矩陣慣例：

```text
Start = j
代表目前已經有 j 個 B1
剩下 15-j 個箱子會重新開
```

如果這一輪結束後變成 i 個 B1，那代表這輪新增了：

```text
i - j 個 B1
```

因為只有剩下的 $15-j$ 個箱子會重開，所以轉移機率是：

$$
P(\text{End}=i \mid \text{Start}=j)
= \binom{15-j}{i-j}p^{i-j}(1-p)^{(15-j)-(i-j)}
$$

其中：

```text
0 <= j <= i <= 15
```

如果 i < j，機率為 0，因為已收集到的 B1 不會消失。

## 用矩陣理解多輪結果

把每一輪從目前狀態到下一輪狀態的機率整理成一張表，就得到一個轉移矩陣。這裡採用線代課本常見的 column-stochastic 寫法：

```text
M[i,j] = P(next = i | current = j)
       = 從 j 個 B1 變成 i 個 B1 的機率
```

也就是：

```text
欄 j = 目前狀態 current = j
列 i = 下一輪狀態 next = i
```

所以矩陣的每一欄都是一個機率向量，每一欄加總為 1。

第一輪從 0 個 B1 開始，所以初始分布是：

```text
v0 = [1, 0, 0, ..., 0]^T
```

每跑一輪：

```text
v_next = M × v_current
```

跑 5 輪後：

```text
v5 = M^5 × v0
```

這個 v5 就是：

```text
5 輪結束後，最後得到 0~15 個 B1 的機率分布。
```

Excel 中可以用 `MMULT` 做矩陣乘法；若採用線代課本常見慣例，矩陣第 j 欄是「目前在狀態 j 時，下一輪會轉到各狀態的機率分布」，所以每一欄加總為 1。

## 為什麼這對遊戲數學重要

這類計算常出現在：

```text
Bonus 遊戲收集指定圖案
Pick bonus 開箱
Hold-and-respin 類機制
保留中獎物件，其餘位置重抽
多輪 feature 的最終獎勵分布
```

單輪可以用二項分配。

多輪且狀態會保留時，應該改成：

```text
二項分配 + 狀態轉移矩陣
```

也就是：

```text
每一輪新增幾個目標圖案，用二項分配算。
多輪累積結果，用轉移矩陣推進。
```

## 精簡重點

二項分配適合處理：

```text
固定 n 次試驗
每次成功機率相同
每次結果可簡化成成功 / 失敗
想知道成功 k 次或某個區間次數的機率
```

在 Excel 中：

```excel
=BINOM.DIST(k, n, p, FALSE)
```

用來算剛好 k 次。

```excel
=BINOM.DIST.RANGE(n, p, lower, upper)
```

用來算區間次數。

但要注意：

```text
同一批箱子中的多種圖案個數彼此不是獨立的。
多輪保留/重開機制不能只看單輪，要用狀態轉移處理。
```
