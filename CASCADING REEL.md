# CASCADING REEL

Source: https://ezslotdesign.com/cascading/

## 核心概念

Cascading Reels 又常被稱為 Tumbling Reels 或 Avalanche。

它的核心不是「一次 Spin 只結算一次」，而是：

```text
Spin 產生盤面
→ 判斷是否中獎
→ 中獎圖案消除
→ 上方圖案掉落補空位
→ 重新判斷是否中獎
→ 重複直到沒有中獎
```

因此，一次付費 Spin 裡可能發生多次得分。

## 和一般 Slot 的差異

一般 Slot：

```text
一次 Spin → 一次盤面 → 一次結算
```

Cascading Reels：

```text
一次 Spin → 多次盤面演化 → 多次結算
```

所以數學上要把一把遊戲視為一個「回合 round」，而不是只看第一個盤面。

```text
Round Win
= 第 1 次盤面得分
+ 第 2 次 cascade 得分
+ 第 3 次 cascade 得分
+ ...
直到 cascade 結束
```

也就是：

```text
E[Round Win]
= E[Win_1 + Win_2 + Win_3 + ...]
```

RTP 的分母仍然是玩家這一把 Spin 的下注額，但分子要包含同一把 Spin 裡所有 cascade 的總得分。

## 計算時要注意什麼

Cascading Reels 的重點不是只算「第一盤中獎機率」，而是還要估計：

```text
1. 第一盤命中率
2. 命中後會消掉哪些格子
3. 補下來的新圖案分布
4. 下一盤再次命中的機率
5. 平均會連消幾次
6. 每次連消的平均得分
```

因此可以把它想成一棵機率樹：

```text
初始盤面
├─ 沒中獎：回合結束
└─ 中獎：給分、消除、補圖案
   ├─ 補完後沒中獎：回合結束
   └─ 補完後再次中獎：繼續給分、消除、補圖案
```

實務上若規則複雜，通常會用模擬或狀態遞迴，而不是手算所有盤面。

## 設計上的重點

Cascading Reels 的玩家期待來自「可以一直連消」。

所以設計上要特別注意：

```text
連消率要夠高
命中率通常要比一般遊戲更友善
賠率可以相對壓低
畫面、音效、節奏要讓連消感覺有累積感
```

如果連消很難發生，玩家會覺得特色沒有發揮出來。

文章中特別提到，連消 4 次有時可能比進免費遊戲還困難，這會讓玩家期待落空。

## 常見進化 1：Progressive Wild

Progressive Wild 的想法是：

```text
連消時，額外把某些圖案轉成 Wild
```

目的：

```text
增加下一次 cascade 再中獎的機率
讓玩家感覺盤面正在變強
建立「再消一次就更接近大獎」的期待
```

這類設計會讓 cascade 不只是重複掉落，而是有明確進度。

例如每連消幾次，就把某一種圖案轉成 Wild；轉換越多，後續盤面越容易再中。

## 常見進化 2：Exploding Reels

Exploding Reels 的想法是：

```text
中獎時，不只消除中獎格子，而是讓相關輪子重轉或替換
```

目的也是增加連消機率。

和一般 cascading 相比，它不是只補空格，而是讓更多盤面位置重新產生變化，因此可能製造更多再次中獎的機會。

## Sevens High 案例

文章提到的 Sevens High 把 cascading 的概念再擴大：

主遊戲：

```text
任意連線得分即可重轉
重轉中若再次得分，就繼續重轉
直到沒有得分才結束
```

並且：

```text
每連續重轉 2 次
→ 將 1 種顏色的 7 轉成 Wild
→ 最多可轉換 3 種顏色
```

免費遊戲：

```text
每 2 把免費遊戲中有得分
→ +2 免費遊戲
→ 並將 1 種顏色的 7 轉成 Wild
```

這個設計把 cascade、re-spin、progressive wild、免費遊戲成長感結合在一起。

## 精簡重點

Cascading Reels 的核心是：

```text
一次下注，可以在同一回合內多次結算。
```

數學上要算的是：

```text
一個 paid spin 觸發多少次盤面結算
以及每次盤面結算的平均得分
```

所以 RTP 可以理解成：

```text
RTP
= E[同一 paid spin 內所有 cascade 得分總和] / Bet
```

特色設計上最重要的是：

```text
連消率
連消後的盤面成長感
連消時的視覺與音效回饋
```
