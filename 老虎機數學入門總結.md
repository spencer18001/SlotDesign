## PART II

FG Retrigger 特色的 RTP 計算。

```text
Total RTP
= Base Game RTP + Free Game RTP
```

```text
Free Game RTP
= 觸發 FG 的機率
  × 觸發後平均總 FG spins
  × 單次 FG spin RTP
```

若 3S / 4S / 5S 給的初始 spins 不同：

```text
Free Game RTP
= (
    P(3S) × E[FG spins | 3S trigger]
  + P(4S) × E[FG spins | 4S trigger]
  + P(5S) × E[FG spins | 5S trigger]
  )
  × FG spin RTP
```

## PART III

PART III 的重點是：同一條中獎線可能同時符合多種獎項時，要取最高分。

例如：

```text
W W W K K
```

這條線可以同時視為：

```text
K 5 連線
W 3 連線
```

實際派彩應取：

```text
MAX(K 5 連線賠率, W 3 連線賠率)
```

為了讓 Excel 好計算，partition 要切乾淨。文章的方式可以整理成三種 pattern：

```text
1. K 連線：
   只會拿 K 獎。

2. K 連線或 W 連線：
   同時可能符合 K 獎與 W 獎，要比較 K 獎與 W 獎，取較高者。

3. W 連線：
   只會拿 W 獎。
```

核心概念：

```text
先把中獎線拆成互斥 partition，
每個 partition 分別計算出現次數，
再判斷該 partition 應該拿哪一個最高獎。
```

## PART IV

Way Game 的核心整理：

```text
1. Way Game 看整個畫面，不看固定 payline。

2. 每一軸可見視窗中，目標圖案或 Wild 出現幾個，
   就算幾個 ways。

3. 只取由左到右連續命中的最長連線，
   不像多線 Line Game 會同時算較短線。

4. 賠表分數是以一把遊戲的 coin / bet cost 為基準，
   所以期望值或派彩要除以 coin。
```

Way Game 的派彩公式可以寫成：

```text
ways
= 第1軸命中數 × 第2軸命中數 × ... × 第N軸命中數
```

```text
payout
= 賠表分數 × ways × totalBet / coin
```

如果只是算 RTP contribution：

```text
RTP contribution
= 出現機率 × 賠表分數 × ways / coin
```

## PART V

Feature / Free Game 觸發率的核心：

```text
Feature 觸發率
= 先定義事件，
  再把所有會觸發的情況拆成互斥 partition，
  最後乘上各自的獎勵期望值。
```

## PART VI

PART VI 的核心可以整理成：

```text
獨立滾輪 = 每一格都有自己的輪帶。
```

計算邏輯：

```text
仍然從單條 line 出發，
用每格輪帶的符號數計算中獎組合數與期望值。
```

同欄相同輪帶時：

```text
每條 line 的 RTP 相同，只要算一條線即可。
```

同欄不同輪帶時：

```text
每條 line 的 RTP 可能不同，必須逐條計算後取平均。
```
