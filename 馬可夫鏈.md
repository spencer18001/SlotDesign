# 馬可夫鏈

Source: https://ezslotdesign.com/markovchain/

## 核心概念

馬可夫鏈（Markov Chain）用來描述一個系統在不同狀態之間轉移的機率。

它最重要的假設是：

```text
下一個狀態只取決於目前狀態，不取決於更早以前的歷史。
```

也就是：

$$
P(X_{t+1}=i \mid X_t=j, X_{t-1}, X_{t-2}, \ldots)
=
P(X_{t+1}=i \mid X_t=j)
$$

這個性質常被稱為 **Markov property**。

在遊戲數學裡，只要 feature 有「狀態會保留、下一輪根據目前狀態繼續變化」的特性，就很適合用馬可夫鏈處理。

常見例子：

```text
Sticky Wild
State Game
Hold and Respin
收集型 bonus
多輪重開 / 保留機制
Free Game 中盤面狀態會延續的特色
```

## 狀態 State

使用馬可夫鏈前，要先定義狀態。

狀態可以是：

```text
目前有幾顆 Sticky Wild
目前收集到幾個指定圖案
目前 meter 在第幾格
目前 Free Game 處於哪一種模式
目前盤面屬於哪一種分類
```

重點是：狀態要足夠描述下一輪的機率。

如果知道目前狀態後，仍然需要更早以前的資訊才能決定下一輪機率，代表狀態定義不夠完整。

## 轉移矩陣

把狀態之間的轉移機率整理成矩陣，就得到轉移矩陣。

這裡採用線性代數常見的 column-stochastic 寫法：

```text
M[i,j] = P(next = i | current = j)
```

意思是：

```text
從狀態 j 轉移到狀態 i 的機率
```

所以：

```text
欄 j = 目前狀態 current
列 i = 下一輪狀態 next
```

矩陣的每一欄都是一個機率分布，因此每一欄加總要等於 1。

```text
sum_i M[i,j] = 1
```

## 狀態向量

如果系統目前可能落在多個狀態，可以用狀態向量表示。

例如有三個狀態：

```text
S1, S2, S3
```

狀態向量可以寫成：

```text
v = [P(S1), P(S2), P(S3)]^T
```

如果目前一定在 S2：

```text
v0 = [0, 1, 0]^T
```

跑一輪後：

```text
v1 = M × v0
```

跑 n 輪後：

```text
vn = M^n × v0
```

這就是馬可夫鏈最基本的計算方式。

## 文章例子：猜拳策略

文章用猜拳作為簡單例子。

甲的策略是固定隨機：

```text
剪刀 30%
石頭 33%
布   37%
```

乙的策略是：

```text
上一場贏：下一場繼續出同一種拳
上一場輸或平手：下一場改出另外兩種拳之一，各 50%
```

因此乙下一把要出什麼，只取決於乙目前這把出什麼，以及這把對甲的輸贏機率。

這正符合馬可夫鏈：

```text
下一個狀態只取決於目前狀態
```

## 建立轉移矩陣

把乙的出拳視為狀態：

```text
S = 剪刀
R = 石頭
P = 布
```

如果乙目前出剪刀：

```text
乙剪刀贏甲布，機率 37%
所以乙下一把繼續剪刀的機率 = 37%
其餘輸或平手機率 = 63%
改出石頭或布，各 31.5%
```

如果乙目前出石頭：

```text
乙石頭贏甲剪刀，機率 30%
所以乙下一把繼續石頭的機率 = 30%
其餘 70% 改出剪刀或布，各 35%
```

如果乙目前出布：

```text
乙布贏甲石頭，機率 33%
所以乙下一把繼續布的機率 = 33%
其餘 67% 改出剪刀或石頭，各 33.5%
```

採用 column-stochastic 寫法，狀態順序為：

```text
[剪刀, 石頭, 布]
```

轉移矩陣為：

$$
M =
\begin{bmatrix}
0.37  & 0.35 & 0.335 \\
0.315 & 0.30 & 0.335 \\
0.315 & 0.35 & 0.33
\end{bmatrix}
$$

矩陣讀法是：

```text
第 1 欄 = 目前出剪刀時，下一把變成 [剪刀, 石頭, 布] 的機率
第 2 欄 = 目前出石頭時，下一把變成 [剪刀, 石頭, 布] 的機率
第 3 欄 = 目前出布時，下一把變成 [剪刀, 石頭, 布] 的機率
```

每一欄加總都等於 1。

## 算第五把出拳分布

文章問題是：

```text
假設乙第一把出石頭，則第五把乙出拳分布為何？
```

第一把已知是石頭，所以：

```text
v1 = [0, 1, 0]^T
```

第五把距離第一把經過 4 次轉移，因此：

```text
v5 = M^4 × v1
```

用上面的矩陣計算，約為：

```text
剪刀：35.2075%
石頭：31.6869%
布：  33.1056%
```

這個結果不是直接把單輪機率乘 5，而是每一輪都根據上一輪的狀態分布再轉移一次。

## 和二項分配的差異

二項分配適合處理：

```text
每次試驗互相獨立
每次成功機率固定
只關心成功次數
```

馬可夫鏈適合處理：

```text
每一輪不是完全重置
目前狀態會影響下一輪機率
需要追蹤狀態分布如何演化
```

所以多輪重開、保留、累積型 feature，通常不能只用單輪二項分配直接算完。

如果狀態會延續，就應該改成：

```text
定義狀態
建立轉移矩陣
用矩陣乘法推進狀態分布
最後對各狀態的獎勵取期望值
```

## 在 Slot RTP 中的用法

假設 feature 有多個狀態：

```text
state 0 = 沒有 Sticky Wild
state 1 = 有 1 顆 Sticky Wild
state 2 = 有 2 顆 Sticky Wild
...
```

每一次 Free Spin 後，狀態可能改變。

可以建立：

```text
M[i,j] = 從 j 顆 Sticky Wild 變成 i 顆 Sticky Wild 的機率
```

然後用：

```text
v_next = M × v_current
```

推進每一輪的狀態分布。

若每個狀態有對應的平均得分：

```text
reward[i] = 處於狀態 i 時，下一 spin 的平均得分
```

則某一輪的期望得分可以寫成：

```text
E[win at round t] = reward^T × v_t
```

整個 feature 的期望值就是把每一輪的期望加總：

```text
Feature EV = sum_t reward^T × v_t
```

如果 feature 有終止狀態，也可以把終止狀態放進矩陣中，讓狀態自然流向終止。

## 精簡重點

馬可夫鏈的核心是：

```text
狀態會延續，但下一輪只看目前狀態。
```

計算流程是：

```text
1. 定義狀態
2. 算出各狀態之間的轉移機率
3. 建立轉移矩陣 M
4. 設定初始狀態向量 v0
5. 用 v_next = M × v_current 推進
6. 若要算 RTP，再把每個狀態的獎勵期望值加權加總
```

和前面的二項分配筆記銜接起來看：

```text
單輪獨立事件：二項分配
多輪狀態延續：馬可夫鏈
```
